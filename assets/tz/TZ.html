Техническое задание №3

Общие требования

  1. Поддержка браузеров: IE9+ .Ghrome ,Opera ,Mozilla ,Safari - 2 последние версии
  2. HTML должен проходить валидацию
  3. Использовать теги HTML5
  4. Использовать modernizr для поддержки HTML5 в старых браузерах
  5. Проверка верстки PerfectPixel'ем. Отклонения 10px+ не допускаются. Исключение - расширение контента ,приводящее к изменению высоты элемента ,а так же растяжение фона
  6. При изменении контента верстка не ломается. Добавляем больше текста в текстовые блоки. Добавляем больше текста в заголовки. Дублируем блоки с контентом. Добавляем новые пункты меню/списка
  7. При изменении размеров окна браузера верстка не должна ломаться. Не должно быть горизонтальных скроллов на стандартном разрешении ,под которое заточен макет
  8. Правильная структура файлов: картинки в папке img, стили в папке css ,скрипты в папке js
  9. Все элементы ,с которыми предполгается взаимодействие ,имеют hover и active - эффекты
  10. Небольшие картинки объединять в спрайты. Там, где можно, использовать иконочные шрифты. По возможности оптимизировать графику
  11. У всех тегов img заполнять атрибуты размеров
  12. Слайдеры и прочие интерактивные элементы верстать с помощью js-библиотек и готовых велосипедов. Хардкорщики могут попробовать на чистом js
  13. Запрещено использование css-фреймворков
  14. Шрифты подключаем через Google Fonts
  15. Указывать min-width в body

HTML и CSS

  1. Крупные блоки сопровождаются комментариями в css ,в html комментарии не нужны
  2. Семантичная разметка. Использовать теги по их прямому назначению. Особое внимание уделить тегам p и h1,h2,h3.. ,h6 ,а так же спискам
  3. Логичные имена классов. Не использовать транслиты
  4. Классы используем для стилизации ,айдишники для JS
  5. Указывать альтернативные шрифты и тип семейства
  6. Не допускать в CSS вложенности больше 2 селекторов
  7. Использовать reset.css или normalize.css
  8. Используем для font-size em'ы или rem'ы. Для line-height единицы
  9. Не использовать инлайн-стили
  10. Использовать псевдоэлементы before и after для декоративных элементов

Препроцессоры

  1. Использовать любой малоизвестный вам препроцессор ,которым вы не пользовались раньше ,либо не иcпользовали вообще.
    Препроцессоры: LESS, SASS(SCSS), Stylus
  2. Если вы впервые знакомитесь с препроцессорами ,используйте его минимальные возможности: переменные и вложенности
  3. Компиляцию производить с помощью специализированных утилит ,либо систем сборщиков (последнее предпочтительнее). Не использовать скрипт для компиляции на клиенте!
  4. Результирующий CSS подключается одним файлом. Допускается минификация CSS
  5. Для препроцессорных файлов использовать отдельную папку (например для всех файлов с расширением ".less" использовать папку "less")
  6. По возможности разбивать структуру макета на блоки. Для каждого блока использовать свой препроцессорный файл
  7. Использовать понятные названия препроцессорных файлов. Каждый файл должен содержать комментарий ,поясняющий ,какие элементы сайта здесь стилизуются

Дополнительные требования

(так же обязательные)
  1. Все элементы ,имеющие hover-эффекты ,должны иметь анимацию. Тип анимации на ваше усмотрение ,если он не указан
  2. Верстку вылкладывать на хостинг Github
  3. Ссылку на верстку и репозиторий выкладывать в соотвтествующее обсуждение (Третий макет)
  4. Срок выполнения - 7 дней. Работу сдать до 18:00 в пятницу (крайний срок: воскресенье 23.08.15.)
  5. Меню в верхней правой части должно плавно выдвигаться при нажатии на кнопку и так же плавно задвигаться обратно.
    Если навессти на кнопку, она должна слегка затемняться на 15-20% (приветствуется решение с помощью препроцессорных функций типа darken)
  6. В каждом слайдере текст должен быть разным. Учитывать то ,что текста может быть больше ,чем на макете.
  7. Сайт должен тянуться на всю ширину окна. Минимальная ширина - 1170px. При ширине окна браузера в 1366px и более ,фон должен растягиваться
  8. В разделе "WE' RE CRAFTY" все элементы должны иметь ховер-эффекты. Первый выделенный элемент на макете показывает ховер-эффект. Вся область вместе с иконкой ,заголовком и текстом должна быть единой ссылкой. То есть при наведении на эту область можно совершить клик.
  9. В разделе "WE' RE CREATIVE" находится 6 прямоугольных блоков. Каждый из этих блоков должен иметь фон в виде картинки (любой. Ищем в интернете). Картинка не должна изменять пропорции (либо обрезается ,либо тянется в обоих направлениях ,обрезаясь по одному из них). При ширине окна браузера 1366px+ блоки должны растягиваться. Ховер-эффект для каждого блока показан на макете. Так же кликабельна вся область контейнера. Реализовать красивую анимацию при наведении (не плавное появление)
  10. Так же раздел "WE' RE CREATIVE" можно "оживить". При нажатии на кнопку ,отличную от "All" ,производится сортировка. Лишние элементы приобретают фон в виде диоганальных линий (как было показано на макете изначально) и перестают быть кликабельными ,так же у них удаляется ховер-эффект. Данное задание необязательно ,но будет огромным плюсом ,если Вы его реализуете
  11. Учитывайте то ,что в любой момент контент может расшириться в любом месте
